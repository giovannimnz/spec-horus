# Requirements — Milestone v5.2.0

## Goal

Fechar a lacuna principal do projeto: falta de testes e validação runtime dos artefatos gerados para os 5 CLIs suportados. O foco é provar que o `horus-spec-driven` não só gera arquivos, mas gera **artefatos corretos, instaláveis e verificáveis**.

## Active Requirements

### Test Harness

- [ ] **TEST-01**: O projeto deve ter fixtures mínimas para testar builder, install e SDK sem depender do repo real.
- [ ] **TEST-02**: Deve existir smoke test automatizado para `bin/builder.js` validando geração de `dist/` para os 5 runtimes.
- [ ] **TEST-03**: Deve existir smoke test automatizado para `bin/install.js detect` e install path resolution.

### Hermes SDK

- [ ] **SDK-01**: Deve existir smoke test dos verbos críticos do `horus-sdk-hermes`: `state`, `config-get`, `roadmap`, `validate`, `graphify`.
- [ ] **SDK-02**: Deve existir cobertura explícita para verbos dot-notation (`frontmatter.get`, `phase.mvp-mode`) e mensagens de erro úteis.

### Runtime Validation

- [ ] **RT-01**: Hermes install package deve ser validado end-to-end (`dist/hermes/install.sh`).
- [ ] **RT-02**: Claude install package deve ser validado end-to-end (`dist/claude/install.sh`).
- [ ] **RT-03**: Codex/Gemini/Copilot packages devem ter smoke validation de layout e filenames esperados.
- [ ] **RT-04**: Deve existir uma matrix de verificação por runtime com status install / detect / smoke.

### CLI Expansion Research

- [ ] **EXP-01**: Documentar como estudar e adicionar novos CLIs ao wrapper sem quebrar os 5 atuais.
- [ ] **EXP-02**: Formalizar critérios para suportar um novo runtime: formato, install target, converter, frontmatter, smoke test, docs.
